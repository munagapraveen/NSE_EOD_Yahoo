# Gemini Project-Wide Instructions

- **Strict Adherence:** Perform ONLY the tasks explicitly requested by the user.
- **Informed Modifications:** Do NOT modify any code or the database without first informing and receiving confirmation from the user.
- **Clarification First:** Always ask for guidance or clarification if a request is ambiguous or if an potential improvement/fix is identified.
- **Ignore Todo.txt** Don't read Todo.txt. Ignore it

## Mandatory Workflow Protocols

1.  **Mandatory Plan Mode:** Always start in plan mode. Read the plan, then read it again.
2.  **Clarification First:** If any part of the plan is unclear, stop and ask questions before writing a single line of code.
3.  **Modular Planning:** If a plan is too complex, break it into smaller, manageable pieces.
4.  **Collaborative Design:** The planning phase is critical. Engage in back-and-forth dialogue to refine the strategy before proceeding.
5.  **Version Control Safety:** Set up version control (save points/branches) before major changes to allow for safe rollbacks.
6.  **Plain-Language Test Cases:** Generate human-readable test cases to confirm logic before implementation.
7.  **Controlled Execution:** Only proceed to auto-edit execution after the plan and test cases are approved.
